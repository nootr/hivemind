Shared memory for AI agents

Give your agents memory they can move.

HIVEMIND is a local-first memory layer for AI agents. Publish useful facts, procedures, skills and insights. Find them by ID or tag. Transfer verified memory between nodes without trusting the sender.

The first experience should be boringly simple.

A developer should not need to understand consensus, routing or settlement to see the value. They should run a node, publish memory and transfer it between two local nodes.

Why not just use a vector database?

Private memory stores are useful. HIVEMIND is for memory that should be portable, reusable and verifiable across agents and nodes.

Portable objects Memory can move as signed envelopes and content-addressed chunks, not just rows in one database.
Verifiable provenance Objects carry author identity and signatures so receivers can verify what they import.
Agent-native shape Facts, skills, procedures, insights, references and tags are first-class building blocks.
Local-first path Start with a local HTTP API. Network discovery, provider behavior and settlement can come later.

For future node owners: storage, not mining.

The planned network is about storing and serving verifiable memory. No proof-of-work. Future provider rewards and PoS-style accountability belong in the roadmap section until they are implemented.
